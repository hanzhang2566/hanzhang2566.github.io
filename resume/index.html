    医护术前行为管理系统v2

    为了应对不同医院对核心流程的定制化需求，重构了 v1 系统的架构。

    将单体的业务系统拆分为基础服务和核心流程服务。基础服务上云配合租户来处理用户、设备、展示等通用服务，根据每家医院的业务需要来实现定制化流程即可。

    通用服务

    aps 服务

    ==项目角色==

    项目负责人

    ==起始时间==

    2023/03 - 至今

    ==项目描述==

    aps = auto process service, 即自动化处理服务。

    基于 Go + gin + viper 开发的、用来自动化推动研发流程的服务。解决了：

    1. PR 提交上来后，第一时间通知到 reviewer 进行代码评审，并能够推送评论到彼此；
    2. PR merge 通过后， 自动运行流水线；
    3. 通过 commit msg 来自动解决禅道 BUG，并自动绑定到最新的发布版本上；

    ==时序图==

    sequenceDiagram
	actor commiter as 提交人

    participant codeup as 代码库
    participant aps as aps 服务
    actor reviewer as 评审人

    participant zentao as 禅道
    participant flow as 流水线

    
    commiter ->> codeup: 提交 PR
    codeup ->> aps: 通知提交 PR
    aps ->> reviewer: 通知提交 PR
    reviewer ->> codeup: merge PR
    codeup ->> aps: 通知 merge 成功
    aps ->> commiter: 通知 merge 成功
    aps ->> flow: 触发流水线
    aps ->> zentao: 进行 BUG 绑定
    aps ->> zentao: 发布版本
