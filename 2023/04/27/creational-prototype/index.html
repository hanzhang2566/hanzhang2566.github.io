模式描述

原型模式是从一个对象的基础上，创建另外一个相同对象，而不用知道创建细节。

需求

使用原型模式创建简历。

UML 类图

classDiagram

class Cloneable {
	<<interface>>
	clone() Object
}

class Resume {
	String name
	int age
	Experience experience
    clone() Resume
}

class Experience {
	String company
    clone() Experience
}

应用场景

  • 对象创建的数据需要从 RPC、网络、数据库、文件系统等非常慢速的 IO 中读取；

  • 对象创建成本大、不同对象大部分字段都相同;

    最佳实践

推荐使用深拷贝，除非有足够的理由。

深拷贝和浅拷贝

  • 浅拷贝只会复制引用（Java 的 Object 方法是浅拷贝）；
  • 深拷贝除了复制引用，还会复制数据本身；
    • 序列化拷贝；
    • 递归拷贝；

也就是说，浅拷贝和原始对象共享引用数据，而深拷贝是完全独立的对象。
