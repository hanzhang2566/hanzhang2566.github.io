模式描述

工厂方法模式(Factory Method)，定义一个用于创建对象的接口，让子类决定实例化哪一个类，使得一个类的实例化延迟到其子类。

需求

工厂方法实现计算器的加减乘除功能。

UML 类图

classDiagram

class CalcFactory {
	<<interface>>
	getOperator(String opt) Operator
}

class BasicCalcCalcFactory {
    getOperator(String opt) Operator
}

class AdvancedCalcCalcFactory {
    getOperator(String opt) Operator
}

CalcFactory <|-- BasicCalcCalcFactory
CalcFactory <|-- AdvancedCalcCalcFactory

class Operator {
	<<abstract>>
    calc(double n1, double n2)* double
}

class AddOperator {
    calc(double n1, double n2) double
}

class MulOperator {
    calc(double n1, double n2) double
}

Operator <|.. AddOperator
Operator <|.. MulOperator
Operator <.. CalcFactory

class OperatorFactory {
	Operator getOperator(String opt)$ Operator
}

OperatorFactory <.. Operator
OperatorFactory <.. CalcFactory

简单工厂和工厂方法的选择

工厂方法模式是对简单工厂中创建产品进行封装，不同的产品使用不同的工厂进行生产，提供一个统一的生产方法给用户。

当对象的创建逻辑比较复杂，不只是简单的 new ，而是要组合其他类对象，做各种初始化操作的时候，推荐使用工厂方法模式，将复杂的创建逻辑拆分到多个工厂类中，让每个工厂类都不至于过于复杂。
