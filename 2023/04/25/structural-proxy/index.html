代理模式

发表于 2023-04-25 本文字数： 541 阅读时长 ≈ 2 分钟
DATE CHANGELOG
2023年4月25日 init

模式描述

不改变原始类（或叫被代理类）代码的情况下，通过引入代理类来给原始类附加功能。

需求

通过代理模式来实现追求者给被追求者送礼物。

UML 类图

classDiagram
class Gift {
	<<interface>>
	giveDoll():void
	giveFlower():void
}

class PursuerProxy {
	Pursuer pursuer
    giveDoll():void
	giveFlower():void
}

class Pursuer {
	SchoolGril gril
    giveDoll():void
	giveFlower():void
}

class SchoolGril {
	String name
}

Gift <|.. Pursuer
Gift <|.. PursuerProxy
Pursuer <-- PursuerProxy
SchoolGril <-- Pursuer

动态代理

静态代理的缺点：1. 编写的目标代理类太多；2. 当代理类接口改变，所有的代理类都需要进行更新。

使用反射可以完成原始类的动态代理。

最佳实践

接口

原始类能够定义接口，一般都是采用实现接口的方式。例如，Spring AOP。

继承

原始类没有定义接口，也没办法给原始类重新定义接口，一般都是采用继承的方式实现代理。例如，cglib。
